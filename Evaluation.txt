# Score More in Assignments

**Beyond the Output:**
Correct output gets you a pass, explanation gets you a distinction.

**Commenting is Mandatory:**
Every line of code needs a comment explaining the logic (not just what it does, but why).

**Structure Matters:**
Well-organized notebooks with markdown headers.

**Modular Coding (The Game Changer):**

* Using functions
* Using Classes and Objects
* Bonus: Separate files for separate concerns

---

# Time Management & Submission

**The Estimation Factor:**
We (trainers) know exactly how long a task should take.

**Too Fast?**
Submitting in 10 mins when it should take 2 hours raises a red flag (potential plagiarism / AI abuse).

**Too Slow?**
Missing deadlines or submitting just before the cutoff indicates poor planning.

**Ideal Window:**
Spend the estimated time ± a reasonable buffer.

**Deadline Discipline:**
Late submissions increase the risk of reduction marks.

---

# How We Evaluate

**Correctness (40%)**
Does the code run? Does it solve the problem?

**Clarity (30%)**
Are the comments clear?
Is the explanation (in markdown/text) coherent?

**Code Structure (20%)**
Is it modular?
Are there functions/classes?

**Presentation (10%)**
Is the notebook clean?
Are outputs visible?
Is it easy to read?

# The AI Dilemma (Can we use AI?)

**YES, you can use AI (ChatGPT, Copilot, etc.) as a tutor or assistant.**

**The Rule:**
If you use AI, you MUST mention it in a comment or a text cell.

Example:

```
# Refactored this loop using suggestion from ChatGPT
```

**The Ban:**
Copy-pasting blindly without understanding.

**Why Honesty matters:**
It builds trust. Interviewers might ask you to explain the AI-generated code.

---

# Grace Marks & Extra Credit

**When do we give extra marks?**

**Exceptional Modularity:**
Going above and beyond the requirement
(e.g., adding error handling, logging).

**Creativity:**
Solving the problem in an innovative, efficient way.

**Documentation:**
Writing a mini "README" or detailed docstrings for your classes in markdown cell.

**Identifier explanation:**
Glossary of what that variable, function, or class will do.

**Grace Marks:**
Reserved for edge cases (technical issues during submission) but not a guarantee.

# Rating Code Quality (The Rating Scale)

**1 Star:**
Spaghetti code, no comments, runs by accident.

**2 Stars:**
Runs correctly, but no comments, hard to read.

**3 Stars:**
Runs correctly, basic comments, monolithic script.

**4 Stars:**
Runs correctly, excellent comments, functional programming.

**5 Stars:**
Production-ready

* Modular (Classes/Modules)
* Excellent comments
* Error handling
* Easy to read and extend
